本文說明如何準備 CSV 檔案，透過 Synx 的匯入功能將交易資料批次匯入帳戶。

---

## 1. 匯入前置條件

> ⚠️ 匯入前請確認以下事項，否則資料將無法正確匯入

### 帳戶必須事先建立

CSV 中所有出現的帳戶名稱（`Account`、`To Account`、`Funding Account`）**必須已在 Synx 中建立**，且名稱須完全吻合（包含大小寫與空格）。匯入時系統不會自動建立新帳戶。

- **投資交易 CSV**：`Account` 必須是**投資帳戶**；`Funding Account` 若填寫，則必須是**非投資帳戶**
- **一般交易 CSV**：`Account` 與 `To Account` 必須是**非投資帳戶**

### 帳戶開始日期須早於 CSV 資料日期

每筆交易的日期必須**晚於**對應帳戶的開始日期，否則該筆資料會被系統拒絕。請在建立帳戶時，確認帳戶起始日期已設定在最早一筆匯入資料之前。

### 投資帳戶須已設定持倉標的

若匯入投資交易，目標投資帳戶中**必須已建立對應的持倉（position）**，即該股票代號（Symbol）已在帳戶中設定。若持倉不存在，該列資料將在寫入時失敗。

### 帳戶不可為已關閉狀態

已關閉的帳戶無法接受新交易匯入，對應列資料將被跳過。

---

## 2. 投資交易 CSV 格式（Investment）

適用於股票買賣記錄。每筆記錄對應一次買入或賣出操作。

> 💡 **帳戶名稱即為股票代號**
> 投資帳戶的名稱直接使用股票代號（如 `TSLA`、`0050`、`BTC-USD`），CSV 中不需要額外的 Symbol 欄位。

### 欄位說明

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| `Account` | ✓ | 投資帳戶名稱（即股票代號），須與 Synx 中完全一致 |
| `Funding Account` | | 對應的資金帳戶（非投資帳戶）；填寫後系統會自動建立對應的資金進出記錄。可留空 |
| `Date` | ✓ | 日期格式：`yyyy-MM-dd`（例如 `2024-01-10`） |
| `Shares` | ✓ | 股數，正數 = 買入，負數 = 賣出。**不可為 0** |
| `Price` | ✓ | 每股價格，正數為一般交易；`0` = 拆股或股息 |
| `Amount` | ✓ | Funding Account 實際增減的金額，**永遠填正數**；拆股或股息時填 `0` |
| `Tax` | | 稅金或手續費，填 `0` 表示無；**不可留空** |
| `Note` | | 備註，可留空；永遠是最後一欄 |

### 買賣判斷邏輯

| Shares | Price | 操作 |
| --- | --- | --- |
| 正數 | 正數 | 買入（Buy） |
| 負數 | 正數 | 賣出（Sell） |
| 正數或負數 | `0` | 拆股 / 股息 |
| `0` | 任意 | 錯誤，該列將被跳過 |

> ⚠️ **Amount 是 Funding Account 的實際增減金額**
>
> `Amount` 填寫的是 **Funding Account 實際變動的金額**，幣別以 Funding Account 為準。若投資帳戶與資金帳戶幣別不同（例如用台幣帳戶買美股），`Amount` 應填台幣金額，而非股票的美元市值。因此 `Amount` 不一定等於 `Shares × Price`。

### 範例

```csv
Account,Funding Account,Date,Shares,Price,Amount,Tax,Note
TSLA,Cash,2024-01-10,100,600,60000,0,
0050,Cash,2024-01-20,-50,650,32500,150,
BTC-USD,,2024-02-01,10,185,1850,0,
```

**範例說明：**

- 第 1 列：買入 TSLA 100 股（Shares 正數），每股 600，總成本 60,000，從 Cash 帳戶扣款
- 第 2 列：賣出 0050 共 50 股（Shares 負數），每股 650，入帳 32,500，稅金 150，存入 Cash 帳戶
- 第 3 列：買入 BTC-USD 10 單位，無對應資金帳戶（僅記錄投資帳戶）

## 3. 非投資交易 CSV 格式（Non-Investment）

適用於一般收支與帳戶之間的轉帳記錄。

### 欄位說明

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| `Account` | ✓ | 非投資帳戶名稱，須與 Synx 中完全一致 |
| `To Account` | | 轉帳目標帳戶名稱（非投資帳戶）；有填寫即視為轉帳交易，留空則依 Amount 判斷為收入或支出 |
| `Date` | ✓ | 日期格式：`yyyy-MM-dd`（例如 `2024-01-15`） |
| `Name` | ✓ | 交易名稱或描述 |
| `Amount` | ✓ | 正數 = 收入，負數 = 支出。**不可為 0** |
| `Note` | | 備註，可留空；永遠是最後一欄 |

### 交易類型判斷邏輯

- `To Account` 有填寫 → **轉帳**
- `To Account` 為空，`Amount > 0` → **收入**
- `To Account` 為空，`Amount < 0` → **支出**
- `Amount = 0` → 錯誤，該列將被跳過

### 範例

```csv
Account,To Account,Date,Name,Amount,Note
Cash,,2026-01-15,Salary,50000,January
Cash,,2026-01-20,Groceries,-1200,
Cash,Savings,2026-01-25,Transfer,5000,
```

**範例說明：**

- 第 1 列：現金帳戶收入 50,000（薪水）
- 第 2 列：現金帳戶支出 1,200（超市消費）
- 第 3 列：從現金帳戶轉帳 5,000 至儲蓄帳戶

## 4. 常見錯誤與處理方式

| 錯誤情況 | 系統行為 |
| --- | --- |
| 缺少必填欄位標題 | 整份 CSV 中止解析，不匯入任何資料 |
| 帳戶名稱不存在 | 跳過該列，顯示錯誤訊息 |
| 帳戶類型不符（如用投資帳戶填一般欄位） | 跳過該列，顯示錯誤訊息 |
| 日期格式錯誤（非 `yyyy-MM-dd`） | 跳過該列，顯示錯誤訊息 |
| Amount 為 0（一般交易） | 跳過該列：「Amount cannot be zero」 |
| Price = 0、Amount = 0（投資交易） | 視為拆股或股息，正常匯入 |
| Shares 為 0 或負數（投資） | 跳過該列：「Shares must be greater than zero」 |
| 帳戶已關閉 | 跳過該列，顯示帳戶已關閉 |
| 投資帳戶無對應持倉 | 寫入失敗，計入失敗筆數 |
| 所有列均有錯誤 | 匯入按鈕停用，無法執行匯入 |
| Share 在賣出後小於 0 | 不允許 Share 小於 0，匯入失敗 |

匯入前系統會顯示預覽確認視窗，列出可匯入的筆數與錯誤明細，確認後才執行實際寫入。
